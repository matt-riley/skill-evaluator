## Dependency notes

Cycle 1 (001–006):

- **002 before 001?** Plan 001 is foundational and can run first, but 002's validation tests are easier to write after 001. Either order works; the table lists 001 first to establish the test harness.
- **001 must precede 005** because the lockfile logic benefits from a working test command and new `workspace_test.go` coverage.
- **001 strongly recommended before 003** so the matcher logic has a place to live in `grader_test.go`.
- **004 is independent** but should avoid conflicting with flag-parsing changes in other plans; run it after 002 if 002 touches `main.go` flag handling.
- **006 is independent** but reads `iteration-N/benchmark.json`; if 005's lockfile is in place, 006 should skip iterations whose lock status is `"running"`.

Cycle 2 (007–016):

- **007 first for value, not for order.** Nothing depends on it structurally, but until it lands, benchmarks over imported evals grade replay noise — every other improvement measures against that noise floor.
- **008 must precede 014** — bundles and `--all-sessions` sync workflows re-import the same steps; without dedupe they duplicate the corpus.
- **009 must precede 013** — activation evals reuse 009's `parseSkillMD` frontmatter parser.
- **010, 012, and 014 all touch `internal/agit/convert.go` signatures** (`ConvertSteps` params in 010, `buildAssertionsWithSignals` params in 012, conversion reuse in 014). Land them in numeric order to avoid rebase churn; each plan notes the collision.
- **011 before 015 if both are scheduled** — 015 writes per-run artifacts and adopts 011's `run-<r>/` layout via `runConfigPath`; landing 015 first means reworking its paths later.
- **012 before 016 is soft** — 016's authoring prompt offers transcript matchers only if 012's prefixes exist (it feature-detects), so either order works but 012-first gives authored assertions more to work with.
- **007 + 014 interact**: bundle imports only carry fixture blobs if 007's restoration reads them through the shared conversion path; 014 adds a test for this once both are in.
- **Probe-first plans (007, 010, 014, 015)** each start with an agit interface probe and carry STOP conditions if the installed agit differs from the `docs/format/*.md` specs (written against agengit v1.26.0). Re-verify probes if agit has been upgraded since 2026-07-01.
