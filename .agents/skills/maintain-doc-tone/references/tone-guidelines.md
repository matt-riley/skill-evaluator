# Tone Guidelines

This repository uses a **Bubbly, Enthusiastic, and Playful** tone for all of its documentation. The goal is to make the tool feel approachable, fun, and easy to use.

## Core Tenets

1. **Be Enthusiastic:** Use exclamation marks (tastefully) and celebrate successes.
2. **Be Conversational:** Write as if you are pair-programming with a friend. Use contractions (it's, you'll, we've).
3. **Use Emojis:** Use emojis as visual anchors for headings, lists, or important notes (e.g., ✨, 🚀, 💡, 🛠️, 🎨).
4. **Keep it Punchy:** Short paragraphs. Snappy sentences.
5. **Avoid Corporate Jargon:** 
   - 🚫 *Instead of:* "Utilize the CLI to execute the evaluation loop."
   - ✅ *Use:* "Run the magic `skill-eval loop` command to handle the whole cycle!"

## Examples of the Tone

**Good (Enthusiastic & Accessible):**
> "Welcome to **skill-eval**! This friendly little CLI tool helps you automate the eval-driven iteration loop for your AI skills."

> "Getting started is a breeze. Just run these commands:"

**Bad (Robotic & Dry):**
> "Skill-eval is a command-line interface designed to automate evaluation loops."

> "Follow the instructions below to install the dependencies required for operation."
